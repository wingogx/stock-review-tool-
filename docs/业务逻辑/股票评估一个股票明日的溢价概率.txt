2. 可以再“打磨一下”的细节
2.1 技术面 S_technical

你现在的逻辑本质是：

封板越早越高分

开板越多越扣分

换手在 10–20% 最舒服

这几个方向我是认可的，只有两点小建议：

time_score 用字符串时间只是伪代码，工程上记得改成分钟数
建议在文档里写清楚是：

# first_limit_minutes: 从 09:30 开始算的分钟数
if first_limit_minutes <= 0:      # 集合竞价一字
    ...
elif first_limit_minutes <= 30:   # 10:00 前
    ...


不然别人可能真的照着 "10:00" 写 😂

一字板特判你已经写了，我建议保留

if is_one_word and S_technical < 1:
    S_technical = 1


这样可以避免出现：一字板被换手率“误伤”太多的情况。

整体：技术面这块 可以直接用。

2.2 资金面 S_capital

你自己已经意识到「涨停股主力净流出很常见，不能一刀切 -2」，现在这个分档就合理多了 👍

唯一的小提醒：

sealed_ratio 之前你定义过是 sealed_amount / amount * 100（百分比）

现在阈值用的是 10 / 3 / 0.5

建议在文档里补一句例子，帮自己 sanity check 一下，比如：

“当 sealed_ratio = 10 表示封单约为当日成交额的 10%。”

只要你们用历史数据看过这个分布，大概觉得：

10% 以上确实很强

3–10% 属于正常
那这一块就 OK，不用再改。

2.3 题材地位 S_theme

你现在改完之后：

是否在「高标梯队分析」里 → 决定 theme_hot_score 是 0/1/2

梯队状态（complete/normal/alone） → 决定 ladder_score 是 -2/0/2

最后 (theme_hot_score + ladder_score)/2 截断到 [-2,2]

这样的结果大概是：

主线 + 完整梯队：

(2 + 2)/2 = +2 ✅

主线 + 独苗（有主线标签但没跟风）

(2 + -2)/2 = 0 → 有利有弊抵消 ✅

不在前十热门 + 独苗：

(0 + -2)/2 = -1 → 冷门独苗吃 -1 ✅

不在前十热门 + normal 梯队：

(0 + 0)/2 = 0 → 普通题材，普通对待 ✅

这个组合我觉得挺优雅的，既不会过度打压非主线，又能让“冷门独苗”吃到惩罚，可以直接用。

2.4 市场环境 S_market

你把退潮期从 -1 改成 -2，我是赞成的。

理由你也写了：

冰点：大家已经躺平，不怎么接力

退潮：一堆人被套在高位，筹码在高空乱飞 → 对“次日溢价”来说，其实是更差的环境

所以：

"退潮期": -2


是符合“溢价视角”的，写进文档就好。

3. 溢价等级映射 & 上线前要做什么

你现在的映射：

≥6：极高

4–6：高

2–4：偏高

0–2：中性

-2–0：偏低

< -2：低

这在纸面上是合理的，但一定要做一件事：

3.1 必须用历史数据做这两个统计

对若干历史样本（比如全部涨停股，或某段时间）：

看 S_total 分布：

有多少票 ≥6、4–6、2–4……

比例太极端就调阈值（比如极高等级太多 or 太少）。

看每一档对应的“真实次日溢价情况”：

比如：

在所有 S_total >= 6 的样本里，次日 正溢价的比例 是多少？

平均涨幅？中位数？最大回撤？

然后你就可以在产品里写成更“科学”的文案，例如：

「历史上与该分数相近的标的，次日有约 68% 出现正溢价（仅统计结果，不构成未来保证）。」

这样比现在“70%+ / 60–70%”这种纯主观估计更扎实。
